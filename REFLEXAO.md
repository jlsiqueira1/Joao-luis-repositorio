No início, enfrentei bastante dificuldade por nunca ter utilizado o Git antes. Os primeiros commits foram frustrantes — não entendia por que minhas alterações não eram reconhecidas, e só depois percebi que precisava registrar as mudanças com git add antes de commitar. Isso me fez perder bastante tempo no começo, mas no dia seguinte consegui resolver com mais calma e rapidez.
No exercício de criação de conflito, cometi o erro de não fazer git push antes de alterar a branch secundária, o que fez o merge acontecer automaticamente sem gerar o conflito esperado. Precisei voltar à branch original, refazer a alteração e repetir o processo. Na hora de resolver o conflito em si, foi mais simples do que imaginei — como eu sabia qual versão estava correta, bastou apagar a parte errada e finalizar o merge.
Um ponto positivo foi me forçar a usar o terminal em vez de depender só da interface do VS Code. Aprendi também sobre commits semânticos, que facilitam muito entender o histórico de mudanças do projeto, e sobre a convenção de nomes para branches, que evita que membros da equipe trabalhem na mesma coisa sem saber.
Para o futuro, preciso praticar mais para fixar o fluxo do Git, especialmente a diferença entre git add e git commit, que ainda gera um pouco de confusão.
Por fim, apesar de ter começado a atividade achando que seria perda de tempo — afinal, o tema era uma introdução ao maior clube do Brasil —, no final reconheci o valor do que aprendi. Só sugiro reduzir a exigência mínima de palavras, já que o conteúdo se esgota bem antes do limite.
